Создать интернет-магазин БАДов «JACK NUTRITION» по следующей архитектуре: на сайте вообще не будет следов о поситителях 

1. Фронтенд: статика на GitHub Pages (HTML, CSS, JS). Без сервера.
2. База данных: Supabase (PostgreSQL, Storage для фото).
3. Защита цен: Supabase Edge Functions (проверка цен, формирование заказа, аналитика).

---

## Функционал сайта (витрина)

### Каталог товаров
- Адаптивная вёрстка: на мобильных устройствах — 2 колонки.
- Карточка товара в каталоге:
  - Фото (сверху).
  - Бренд, название.
  - Форма выпуска (например, «30 капсул», «60 таблеток», «30 саше», «300 грамм»).
  - Цена слева, кнопка «В корзину» справа снизу.
  - После добавления в корзину: блок с «–», счётчик, «+».
- При клике на карточку — плавающее окно (модальное) с полной информацией:
  - Полное описание, состав, дозировка, способ применения, противопоказания.
- Поиск по названию и бренду (обычный поиск).
- Поиск по штрих-коду: отдельный значок (сканер), при нажатии открывается камера, сканирует штрих-код, ищет товар в базе. Если найден — показывает его. Если нет — сообщение «Товар не найден».
- Фильтры: по категориям, брендам, цене (диапазон).
- Сортировка: по цене (возр./убыв.), по названию, по новизне.

### Корзина
- Хранится в localStorage (только id товара и количество).
- Отображение: список товаров с кнопками «+»/«–» для изменения количества, общая сумма.
- Кнопка «Оформить заказ» — отправляет корзину в Edge Function.

### Оформление заказа
- Фронтенд отправляет в Edge Function: { cart: [{ id, quantity }] }.
- Edge Function:
  - Проверяет все id в базе данных.
  - Берёт актуальные цены.
  - Рассчитывает итог.
  - Формирует текст заказа.
  - Сохраняет обезличенную запись о продаже в таблицу аналитики (без данных клиента).
  - Открывает WhatsApp с текстом заказа.
- Если клиент подменил данные — Edge Function подставляет верные цены и игнорирует подделку.
- При ошибке Edge Function (недоступность БД) — в текст заказа добавляется спецкод (например, «[!CHECK!]»), понятный только продавцу, и заказ всё равно отправляется.
- Защита от двойных нажатий: кнопка блокируется на время отправки.
- Номер заказа: сквозной счётчик, формат: ДДММГГ/XXX (например, 130726/137).

### Ограничение по времени оформления заказа
- Сайт доступен для просмотра 24/7: можно смотреть товары, добавлять в корзину, фильтровать, искать.
- Оформление заказа (кнопка «Оформить заказ») работает ТОЛЬКО в рабочие часы: с 9:00 до 20:00.
- В нерабочее время (с 20:00 до 9:00):
  - Кнопка «Оформить заказ» неактивна (заблокирована или скрыта).
  - При попытке нажатия показывается сообщение: «Заказы принимаются с 9:00 до 20:00. Добавьте товары в корзину и оформите заказ утром.»
- В админ-панели добавить переключатель (включить/выключить) для этого режима:
  - Если режим выключен — заказы можно оформлять круглосуточно.
  - Если режим включён — заказы доступны только с 9:00 до 20:00.
- Время работы определяется по часовому поясу, указанному в настройках магазина (Москва UTC+3).

## Админ-панель

### Доступ
- Логин и пароль (Supabase Auth).
- Кнопка «Забыли пароль?» — сброс на почту.
- Смена пароля внутри админки.
- Доступ по пути /admin (на том же домене).

### Управление товарами (CRUD)
- Список товаров с пагинацией и поиском.
- Добавление/редактирование всех полей:
  - Название, описание (краткое и полное), категория, бренд, цена, старая цена, остаток, объём, артикул, штрих-код.
  - Бейджи: Хит, Новинка, Скидка (флажки).
  - Поля: срок годности, способ применения (дополнительные, необязательные).
- Фото: до 4 штук на товар (загрузкой или по ссылке, комбинированно). Главное фото выбирается вручную.
- Ссылки: до 4 ссылок (видео, инструкция, отзывы) на товар.
- Быстрое изменение количества (добавить на склад).
- Кнопки: скрыть/показать/удалить/дублировать товар.
- Отдельный раздел для штрих-кодов: можно отсканировать камерой или ввести вручную.

### Управление категориями и брендами
- Добавление, переименование, удаление.
- При импорте из Excel — автоматическое создание, если отсутствуют.

### Импорт из Excel
- Поддерживаемые форматы: .xlsx, .xls.
- При совпадении артикула (sku): обновлять все поля. Если поле в Excel пустое — не затирать существующее значение.
- При ошибке в файле — остановить импорт, показать сообщение и указать строку.
- Категории и бренды создаются автоматически, если их нет в базе.

### Экспорт в Excel
- Выгрузка всего каталога (все поля) в .xlsx.

### Статистика продаж (без клиентских данных)
- Таблица orders_analytics: только обезличенные данные (номер заказа, список товаров, сумма, дата).
- Отображение в админке:
  - Общее количество проданных товаров.
  - Общая выручка за всё время.
  - Графики по дням/неделям/месяцам (базовые, Chart.js).
  - Топ-10 товаров по количеству проданных штук (фильтр по периоду: месяц, квартал, год, всё время).
  - Список заказов за выбранный период (номера, товары, суммы).
- Данные обновляются в реальном времени при каждом заказе.

### Настройки магазина
- WhatsApp номер (международный формат).
- Название магазина.
- Логотип (загрузка, ссылка, или текст).
- Валюта рубль
- Шаблон текста заказа (опционально).
- Текст для ошибки проверки (спецкод).
- Часовой пояс (Europe/Moscow по московскому времени) для корректной работы ограничения по времени.
- Переключатель для режима ограничения времени оформления заказа (включить/выключить).

### Резервное копирование
- Кнопка в админке «Скачать дамп БД» (SQL-дамп или JSON).
- На случай перехода на другую БД или восстановления.

### Мониторинг
- Индикатор работы Edge Function (зелёный/серый кружок) в админке и на сайте.
- Уведомления в WhatsApp при падении сайта или функции на тот же номер текст сайт не активен и причина (если возможно).

## Безопасность и анонимность

- Нет кук для идентификации посетителей.
- Нет логирования IP-адресов.
- Нет таблиц с персональными данными клиентов (имя, телефон, город).
- В аналитике — только обезличенные данные о продажах.
- Все расчёты цен на сервере (Edge Function), клиент не может подменить итог.

## Технологии

- Фронтенд: HTML, CSS, JavaScript (без фреймворков, чтобы было легко).
- База данных: Supabase (PostgreSQL, Storage для фото).
- Серверная логика: Supabase Edge Functions (защита цен, аналитика).
- Хостинг статики: GitHub Pages.
- Домен: пока бесплатный поддомен GitHub Pages (username.github.io), админка по пути /admin.
